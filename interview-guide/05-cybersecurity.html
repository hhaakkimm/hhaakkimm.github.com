Vulnerability Management Fundamentals

Core Concepts

  • CVE (Common Vulnerabilities and Exposures): Standardized identifiers for known vulnerabilities (e.g., CVE-2024-1234)
  • CVSS (Common Vulnerability Scoring System): 0-10 scoring system
    • 0.1-3.9: Low
    • 4.0-6.9: Medium
    • 7.0-8.9: High
    • 9.0-10.0: Critical
  • CWE (Common Weakness Enumeration): Categories of software weaknesses (e.g., CWE-79: Cross-site Scripting)
  • OWASP Top 10: Most critical web application security risks
  • Zero-Day Vulnerability: Unpatched vulnerability actively exploited

Vulnerability Lifecycle

  1. Discovery: Automated scanning, manual testing, security research
  2. Assessment: Severity scoring, impact analysis, exploitability
  3. Prioritization: Risk-based ranking considering business context
  4. Remediation: Patching, mitigation, configuration changes
  5. Verification: Confirm fix effectiveness
  6. Reporting: Document and communicate status

Vulnerability Scanning Technologies

SCA (Software Composition Analysis)

  • Purpose: Identify vulnerabilities in third-party libraries and dependencies
  • Tools: Snyk, Dependabot, WhiteSource, Black Duck
  • Example:
    • Scan package.json or requirements.txt
    • Match dependencies against vulnerability databases
    • Report vulnerable versions (e.g., Django 3.2.18 has CVE-2023-XXXX)

SAST (Static Application Security Testing)

  • Purpose: Analyze source code for security flaws without execution
  • Tools: SonarQube, Checkmarx, Fortify, Bandit (Python)
  • Detects: SQL injection, XSS, hardcoded secrets, insecure crypto

DAST (Dynamic Application Security Testing)

  • Purpose: Test running applications from the outside (black-box)
  • Tools: OWASP ZAP, Burp Suite, Acunetix, Nessus
  • Detects: Authentication issues, configuration errors, runtime vulnerabilities

IAST (Interactive Application Security Testing)

  • Purpose: Monitor from within during testing (instrumentation)
  • Tools: Contrast Security, Checkmarx IAST
  • Advantage: Better accuracy, real-time feedback

Container & Infrastructure Scanning

  • Container Scanning: Trivy, Clair, Anchore
    • Scan Docker images for OS and application vulnerabilities
    • Check for misconfigurations
  • Infrastructure: Qualys, Rapid7, Tenable
  • Cloud Security: AWS GuardDuty, CloudDefense, Prisma Cloud

Threat Intelligence

Key Concepts

  • TTPs (Tactics, Techniques, Procedures): How attackers operate
  • IOCs (Indicators of Compromise): Evidence of breach (IPs, hashes, domains)
  • MITRE ATT&CK Framework: Knowledge base of adversary tactics
    • Initial Access ‚Üí Execution ‚Üí Persistence ‚Üí Privilege Escalation
    • Defense Evasion ‚Üí Credential Access ‚Üí Discovery ‚Üí Lateral Movement
    • Collection ‚Üí Exfiltration ‚Üí Impact
  • Threat Feeds: AlienVault OTX, CISA, NVD (National Vulnerability Database)

Security Standards & Compliance

Major Frameworks

  • NIST Cybersecurity Framework: Identify, Protect, Detect, Respond, Recover
  • ISO 27001: Information security management standards
  • PCI DSS: Payment card industry security standards
  • SOC 2: Trust service criteria for cloud providers
  • GDPR: Data protection and privacy (EU)

Integration with 150+ Security Tools

Common Integration Patterns

  • REST API Integration: Poll for vulnerability data
  • Webhooks: Real-time notifications on new findings
  • File-based: CSV/JSON exports, SFTP transfers
  • SIEM Integration: Splunk, ELK Stack, QRadar

Data Normalization Challenge

  • Different vendors use different severity scales
  • Varying data formats and schemas
  • Deduplication across multiple sources
  • Correlation of findings to the same underlying vulnerability

Risk-Based Prioritization

Factors to Consider

  1. CVSS Score: Base severity
  2. Exploitability: Is exploit code publicly available?
  3. Asset Criticality: Production vs development
  4. Exposure: Internet-facing vs internal
  5. Threat Intelligence: Is it being actively exploited?
  6. Business Impact: Financial, reputation, regulatory

Example Scoring Algorithm

Risk Score = (CVSS * 0.3) 
           + (Exploitability * 0.25) 
           + (Asset Criticality * 0.25) 
           + (Exposure * 0.15) 
           + (Active Threats * 0.05)

OWASP Top 10 (2021)

  1. Broken Access Control: Unauthorized access to resources
  2. Cryptographic Failures: Weak or missing encryption
  3. Injection: SQL, NoSQL, LDAP, OS command injection
  4. Insecure Design: Missing or ineffective security controls
  5. Security Misconfiguration: Default settings, verbose errors
  6. Vulnerable & Outdated Components: Unpatched libraries
  7. Identification & Authentication Failures: Weak passwords, session management
  8. Software & Data Integrity Failures: Insecure CI/CD, unsigned updates
  9. Security Logging & Monitoring Failures: Insufficient logging
  10. Server-Side Request Forgery (SSRF): Forced requests to unintended locations
